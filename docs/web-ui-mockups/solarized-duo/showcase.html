Add Stripe webhook handler

Implement an endpoint that receives Stripe webhook events, verifies signatures, and writes idempotent records to the payments ledger. This unblocks reconciliation work in the Payments revamp epic.

Acceptance criteria

  • Verify the Stripe-Signature header against the signing secret.
  • Handle payment_intent.succeeded and charge.refunded.
  • Persist events idempotently keyed on the Stripe event id.

Sketch

// verify + dispatch pub async fn handle_webhook(req: Request) -> Result<Response> { let evt = stripe::verify(&req, SIGNING_SECRET)?; match evt.kind { "payment_intent.succeeded" => ledger::record(evt).await?, "charge.refunded" => ledger::reverse(evt).await?, _ => {} } Ok(Response::ok()) }

Checklist

  • Scaffold route + handler
  • Signature verification
  • Idempotency store (waiting on #45)
  • Integration test with Stripe CLI
