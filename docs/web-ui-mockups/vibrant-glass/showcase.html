Add Stripe webhook handler

Summary

Implement a robust handler for incoming Stripe webhook events so payment state stays consistent with the source of truth. Must verify signatures and dedupe via the idempotency key.

Acceptance criteria

  • Verify the Stripe-Signature header against the signing secret.
  • Persist raw event payloads for replay.
  • Handle payment_intent.succeeded and charge.refunded.

Example handler

async fn handle_webhook(req: Request) -> Result<Resp> {
    let event = stripe::verify(&req, SECRET)?;
    match event.kind {
        Kind::PaymentSucceeded => ledger.commit(event).await?,
        _ => tracing::info!("ignored {}", event.kind),
    }
    Ok(Resp::ok())
}

Tasks

  • Scaffold route + signature verify
  • Persist raw payloads
  • Dedupe by idempotency key (waiting on #45)
  • Backfill replay tooling
