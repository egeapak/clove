#42 FFeature ○ Open ↑ High · P1
Add Stripe webhook handler

Implement a robust webhook endpoint for Stripe events so payment state stays in sync with our ledger. This unblocks the rest of the Payments revamp epic.

Background

Stripe delivers asynchronous events (payment_intent.succeeded, charge.refunded, …). We must verify signatures and process each event idempotently — depends on the idempotency fix in #45.

Requirements

  • Verify the Stripe-Signature header against the signing secret
  • Persist raw events before processing (replay-safe)
  • Return 200 within 5s; defer heavy work to a queue

Reference handler

pub async fn handle_webhook(req: Request) -> Result<Response> {
    let sig = req.header("Stripe-Signature")?;
    let event = stripe::verify(&body, sig, SECRET)?;
    // idempotent: skip if event.id already seen
    store.record(event).await?;
    Ok(Response::ok())
}

Acceptance

  • Endpoint scaffolded & routed
  • Signature verification
  • Idempotent event processing (waits on #45)
  • Integration tests with Stripe CLI fixtures
