#42 F feature ○ open ↑ p1
Add Stripe webhook handler

Receive and verify Stripe webhook events so subscription state stays in sync. Must be idempotent and tolerate retries from Stripe's at-least-once delivery.

Acceptance criteria

  • Verify Stripe-Signature using the endpoint secret
  • Persist the raw event before processing (replay safety)
  • Return 2xx within 5s; defer heavy work to a queue

Handler sketch

pub async fn handle(req: Request) -> Result<Response> {
    let ev = stripe::verify(req.body, &secret())?;
    match ev.kind {
        "invoice.paid"     => ledger::credit(ev).await?,
        "customer.deleted" => ledger::revoke(ev).await?,
        _ => {} // ignore the rest
    }
    Ok(Response::ok())
}

Checklist

  • Define event schema
  • Add signature verification
  • Wire up the queue worker
  • Backfill missed events (blocked by #45)
